# 공명문 사고 체계 및 논리 구조화 (Gongmyung Thought Process)

## 1. 공명문의 본질 (Essence of Gongmyungmun)
> "문법 구조는 곧 사고의 방식이다."

공명문은 단순한 디버깅 도구가 아니라, **사고의 흐름(Flow of Thought)**을 시각화하는 언어 체계입니다.
영어의 결과 중심 사고나 한자의 함축적 사고와 달리, **한국어의 조사(은/는/이/가)**처럼 **주체에서 시작하여 세상으로 나아가는 흐름(Flow, ~)**을 명확히 표기합니다.

### 언어별 사고 방식 비교
*   **한국어/공명문 (2인칭적 사고)**: `나(주체) -> 너/세상(객체)`
    *   흐름: **주체(●)**가 **조건(○)**을 만나 **결과(◎)**로 이어짐.
*   **영어**: `결과 <- 주체` (결과가 먼저 오거나 주체가 뒤따름)
*   **한자**: 흐름(조사)의 생략, 의미의 덩어리.

## 2. 공명문 논리 구조 (Logic Structure)
사용자가 제시한 **"주체에서 세상으로 이어지는 사고 흐름"**을 공명문 기호로 구조화합니다.

### 기본 흐름 (Basic Flow)
```gongmyung
●(주체) ~ ○(조건) ~ ◎(결과)
```

### 심화 사고 흐름 (Advanced Thought Loop)
주체가 조건을 만나 결과를 내고, 그 결과에서 추론을 하거나, 실패하여 다시 주체로 돌아가는 순환 구조입니다.

```gongmyung
●(주체) ~ ○(조건) ~ ◎(결과) ~ ∴(추론) ~ [ ◎(최종 결과) || ×(실패) ~ ●(재시도/주체) ]
```

## 3. 공명문 작성 예시 (Examples)

### 예시 1: 글쓰기 (Writing)
> "나는(주체) 영감을 받아(조건) 글을 쓴다(결과)."

```gongmyung
# ●(나) ~ ○(영감 수신) ~ ◎(글 작성)
```

### 예시 2: 프로그래밍 (Programming)
> "함수는(주체) 입력값이 유효하면(조건) 값을 반환하고(결과), 아니면(조건) 에러를 낸다(실패)."

```gongmyung
# ●(함수) ~ [ ○(입력 유효) ~ ◎(값 반환) || ○(입력 무효) ~ ◎(에러) × ]
```

### 예시 3: 이치의 탑 (Tower of Reason)
> "이치의 탑은(주체) 6단계 논리로 정의되며(정의), 사용자가 내용을 채우면(조건), 공명을 일으킨다(결과)."

```gongmyung
# ●(이치의 탑) ≡ [계념-구조-코드-구현-실행-공명] ~ ○(내용 충족) ~ ◎(공명 발생)
```

## 4. 공명 양면책 구조 (Double-Sided Book)
*   **A면 (공명문)**: 위와 같은 **사고의 문법**을 기록하는 주석 시스템. (논리)
*   **B면 (크레이프케이크)**: 이 사고들이 시간 순서대로 쌓여 만들어진 **기억의 지층**. (역사)
