# [GM-2025] ●-System-Initiated
# 공명문 (Gongmyungmun): The Visual Code Philosopher
## "가장 낮은 곳에서, 가장 선명하게 흐르는 논리"

### 1. 서문: 왜 공명문인가? (Why Gongmyungmun?)
우리는 너무 무거운 시대에 살고 있습니다.
코드를 한 줄 읽기 위해 수 기가바이트의 IDE를 설치해야 하고, AI가 쏟아내는 수백 줄의 코드는 블랙박스처럼 불투명합니다.

**"메모장(Notepad) 하나면 충분하지 않은가?"**

이 질문에서 공명문은 시작되었습니다.
오래된 노트북, 인터넷이 끊긴 환경, 1코어의 열악한 상황에서도
우리는 코드의 **흐름(Flow)**과 **맥락(Context)**, 그리고 시스템의 **고통(Load)**을 읽을 수 있어야 합니다.

### 2. 핵심 철학 (Core Philosophy)
1.  **Notepad-Native**: 화려한 GUI 없이, 텍스트 파일 하나로 모든 논리를 시각화한다.
2.  **Rhythm & Flow**: 코드는 정지된 텍스트가 아니라, 흐르는 음악(Beat)이다.
3.  **Symbolic Logic**: 복잡한 영어 문법 대신, 직관적인 기호로 논리를 압축한다.

### 3. 훈민 기호 체계 (The Symbol System)
공명문은 5가지 핵심 기호로 모든 코드의 흐름을 표현합니다.

*   `●` (시작/초기화): 비트가 생성되고, 이야기가 시작되는 지점.
*   `∴` (분기/조건): "그러므로" - 흐름이 갈라지는 선택의 순간 (if/switch).
*   `◎` (반복/루프): 에너지가 순환하며 작업을 수행하는 구간 (for/while).
*   `⇒` (전이/결과): 상태가 변화하여 다음 단계로 나아가는 흐름.
*   `×` (오류/예외): 흐름이 막히거나, 예상치 못한 충돌이 발생한 지점.

### 4. 당신에게 필요한 이유
*   **초보자**: AI가 짜준 코드가 어떻게 돌아가는지 '그림'으로 이해하고 싶을 때.
*   **저사양 개발자**: 무거운 도구 없이도 복잡한 로직을 설계하고 싶을 때.
*   **예술가**: 코드의 리듬과 구조를 시각적인 패턴으로 감상하고 싶을 때.
